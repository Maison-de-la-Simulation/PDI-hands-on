1 # PDI: the Portable Data Interface.
2 
3 PDI is a library that aims to decouple high-performance simulation codes from Input/Output concerns.
4 It offers a declarative application programming interface that enables codes to expose the buffers in which they store data and to notify PDI of significant steps of the simulation.
5 It supports a plugin system to make existing libraries such as HDF5, SIONlib or FTI available to codes, potentially mixed in a single execution.
6 
7 This approach makes it possible to describe the I/O operations in a dedicated YAML file instead of interleaving them with the simulation code and thus to improve their portability and maintainability.
8 The public plugin API offered by PDI is general and simple enough that one can easily add support for the best suited library for its use-case (problem size, IO type, hardware used, etc.)

11 ## Prerequisites
12 
13 PDI depends on:
14  * cmake, version >= 3.5 (version >= 3.10 if you want to run tests)
15  * a C-99 and C++-14 compiler (gcc-5.4 is tested)
16  * a POSIX compatible OS (linux with GNU libc-2.27 is tested)
17  * a MPI-2 library (openmpi-1.10.2 is tested)
18  * paraconf (distributed with PDI, pass the `-DUSE_SYSTEM_PARACONF=ON` option to cmake to use an external one)
19  * libyaml (distributed with paraconf, pass the `-DUSE_SYSTEM_YAML=OFF` option to cmake to use it)
20 
21 PDI Fortran support depends on:
22  * a Fortran-2003 compiler (gfort-5.4 is tested)
23  * BPP (distributed with PDI)
24  * a python and bash interpreter at compilation
25 
26 Plugins:
27  * the decl'HDF5 plugin require a version of HDF5 compatible with the chosen MPI (either sequential or parallel)
28  * the FTI plugin depends on the FTI library that is distributed together with PDI in the `vendor` directory
29  * the decl'SION plugin depends on SIONlib
